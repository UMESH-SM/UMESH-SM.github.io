Securing Files stored on Cloud using Hybrid Cryptography and LSB Image Steganography

Home Project SecureCloud

It focuses on a safer way to store and share files on Cloud.

#1. Abstract

Today cloud computing is used in many areas to store huge amount of data. But there are many issues while storing data on cloud and sharing them.
Cryptography and Steganography are the most commonly used solutions.
But using a single encryption algorithm is not effective to provide high level security to the data in cloud computing.
We thereby need a much more secured system.

This project provides a better security mechanism using Hybrid Symmetric Key Cryptography and Steganography.
This uses AES, DES and RC2 algorithms to provide block wise security to data and LSB Steganography technique to embed the Private Keys used for encryption within a cover image for further security during file sharing.

#2. Objectives

  • To achieve a Secured platform for storing files on Cloud.
  • Enhance the encryption process using three Encryption algorithms to encrypt the files.
  • Enhance the security while file sharing.
  • Provide additional protection while sharing the private keys using LSB Steganography.

#3. Methodology

  • It takes a text file as input from the User.
  • Splits it into 3 equal parts.
  • Encrypt the 3 parts with 3 different Encryption Algorithms.
  • Join them together after encryption is complete and stored it on cloud.

  • It takes the encrypted file on the cloud as input.
  • Split it into 3 parts.
  • Decrypt it using the same 3 algorithms.
  • Join them together after decryption and provide it to the User.

There are 2 scenarios in File Sharing.

  • In the first Scenario, when a User sends a request for a file the server sends an email to notify the request to the respective author of the file.
  • The email consists of the details of the User requesting the file, name of the file being requested and the purpose of requisition.
  • The owner has 2 choices here, either he/she can accept or reject the request.
  • If the owner accepts the request, the secret keys used for the encryption of the requested keys will be embeded into a JPG image using LSB Image Steganography technique and send it to the requester's email.

  • Once the requester recieves the Stego image containing the secret keys he/she can decode it in the Decode Stego Image module.
  • After decoding is complete it gives the secret keys that can used to decrypt the requested file on the sever.
  • In the second Scenario, if the owner rejects the request then the requested user will be notified about the same via email.

#4. Conclusion

  • With the help of Hybrid Encryption Mechanism we are able to increase the security of files stored on cloud.
  • And using Steganography enhances the security while the files are being shared among the Users.
  • Combining both these mechanisms make a much more Secure System to store and share files on cloud.
